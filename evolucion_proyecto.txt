OBJETIVO ALCANZADO:
  ✓ Extracción modular de lógica de indexación hacia librería ontology_rag
  ✓ Implementación del patrón OOP con Inyección de Dependencias
  ✓ Mejora significativa en testabilidad y reutilización de código

CAMBIOS REALIZADOS:

1. NUEVO MÓDULO: src/ontology_rag/creador_rag.py
   ──────────────────────────────────────────────
   Clase: OntologyIndexer
   
   Responsabilidades encapsuladas:
     • Parsing robusto de múltiples formatos RDF (TTL, N3, OWL, RDF, NT)
     • Análisis estructural automático de ontologías (CORE vs EXTENSION)
     • Extracción de Clases y Propiedades mediante SPARQL
     • Fragmentación semántica de documentos
     • Vectorización con HuggingFace Embeddings (configurable)
     • Persistencia en ChromaDB con metadatos estructurales
     • Soporte para procesamiento en lotes (configurable)
   
   Inyección de Dependencias:
     • source_directories: List[str] - Rutas de entrada (múltiples)
     • persist_directory: str - Ruta de salida ChromaDB
     • embedding_model: str - Modelo HF intercambiable
     • batch_size: int - Tamaño de procesamiento configurable
     • logger: Optional[Logger] - Logging customizable
     • recreate_db: bool - Modo regeneración vs. adición
   
   Beneficios:
     ✓ Fácil testabilidad con mocks de dependencias
     ✓ Reutilización en múltiples contextos (CLI, API, scripts)
     ✓ Cambios de configuración sin modificar código
     ✓ Logging inyectable para trazabilidad

2. ACTUALIZACIÓN: src/ontology_rag/__init__.py
   ───────────────────────────────────────────
   • Exportación de OntologyIndexer en __all__
   • Integración transparente con módulo existente

3. NUEVO SCRIPT: code/run_indexacion.py
   ──────────────────────────────────────
   Cliente de indexación que:
     • Define rutas físicas reales (entrada/salida)
     • Instancia OntologyIndexer con inyección explícita
     • Ejecuta pipeline de indexación completo
     • Maneja errores y logging centralizado
   
   Uso:
     $ python code/run_indexacion.py
